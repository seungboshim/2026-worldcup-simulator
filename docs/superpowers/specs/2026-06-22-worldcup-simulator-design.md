# 2026 월드컵 시뮬레이터 — 설계 문서

- **작성일**: 2026-06-22
- **상태**: Phase 1 설계 승인됨
- **대상 대회**: 2026 FIFA 월드컵 (북중미, 48개국 / 12개 조 / 32강 토너먼트)

---

## 0. 배경 & 전체 그림

사용자가 직접 경기 결과를 조절하며 월드컵 전 일정을 시뮬레이션하는 웹 앱.
조별리그는 스코어를 조절하고, 토너먼트(32강~결승)는 승패만 입력한다.

전체 제품은 3단계로 나눈다. **이 문서는 Phase 1만 구현 대상으로 한다.**

| Phase | 범위 | 상태 |
|-------|------|------|
| **Phase 1** | 코어 시뮬레이터 (조별리그 스코어 조절 + 조 순위 + 3위 랭킹 floating + 토너먼트 승패 + 다크모드/녹색 테마). 로컬 전용(localStorage). | **이번 스펙** |
| Phase 2 | Supabase 백엔드. 예측 제출 저장(익명, 브라우저별 ID) + 통계 대시보드(우승 top5, 결승 top5, 4강·8강… 배당률 느낌). | 후속 |
| Phase 3 | 골든볼/골든슈 등 어워드 투표 (선수 데이터 필요). | 후속 |

---

## 1. Phase 1 범위

조별리그(12개 조) 스코어 시뮬레이션 → 32강 진출팀 산출 → 토너먼트(승패만) → 우승까지.

- **백엔드 없음, localStorage 전용.**
- 통계/투표/Supabase/Auth는 Phase 1 범위 밖.
- 디폴트값 = 실제 진행된 2026 월드컵 경기 결과(시드). 언제든 초기화 가능.

---

## 2. 데이터 구조

### 2.1 Seed (정적 JSON, repo에 커밋) — `data/worldcup-2026.json`

- `teams`: `{ id, name, flagCode, groupId }` — 48팀
- `groups`: A~L (12개 조 × 4팀)
- `groupMatches`: 조당 6경기(4팀 라운드로빈) × 12 = **72경기**
  - `{ id, groupId, homeId, awayId, played, defaultHome, defaultAway }`
  - `played=false`면 아직 안 치러진 경기 (디폴트 스코어 없음)
- `knockoutSlots`: R32(16) · R16(8) · 8강(4) · 4강(2) · 결승(1) 구조 정의
  - 각 슬롯이 어떤 조의 몇 위(또는 어떤 3위 배정군)에서 채워지는지 명세
  - **3위팀 → R32 슬롯 배정 매트릭스** 포함 (3.3 참고)

### 2.2 런타임 상태 (localStorage)

- `groupScores`: `{ [matchId]: { home: number, away: number } | null }`
- `knockoutWinners`: `{ [matchId]: winnerTeamId | null }`
- 파생값(조 순위, 3위 랭킹, 32강 진출팀, 브래킷 진행)은 저장하지 않고 순수 함수로 계산.

---

## 3. 핵심 로직 (순수 함수 — 독립 유닛테스트, TDD)

### 3.1 `computeGroupStandings(group, scores)`
4팀을 FIFA 순서로 정렬: 승점 → 득실차 → 다득점.
(승자승/페어플레이/추첨은 Phase 1 범위 밖, 추후.)
팀별 산출: 경기수 P / 승 W / 무 D / 패 L / 득점 GF / 실점 GA / 득실차 GD / 승점 Pts.
미입력(`played=false`) 경기는 순위 계산에서 제외.

### 3.2 `computeThirdPlaceRanking(groups, scores)`
12개 조의 3위팀을 모아 FIFA 순서로 정렬, **상위 8팀에 진출 플래그**.
나머지 4팀은 탈락 플래그.

### 3.3 `seedKnockout(standings, thirdPlaceRanking)`
R32 16경기 대진을 공식 구조로 생성.
2026 포맷의 핵심 난점: "어느 조 3위가 어느 R32 슬롯으로 가는지"가 **진출한 3위팀들의 조 조합**에 따라 결정되는 공식 배정표를 정확히 구현해야 한다.

### 3.4 `advanceBracket(winners)`
각 라운드 승자를 다음 라운드 슬롯으로 전파. 미선택 경기는 빈 슬롯.

> 모든 함수는 부수효과 없는 순수 함수 → Vitest 유닛테스트로 검증.

## 4. UI 레이아웃

```
┌──────────────────────────────────────────┬─────────────┐
│ Header: 타이틀 · [☀/🌙 토글] · [리셋]      │  Floating   │
├──────────────────────────────────────────┤  3위 랭킹    │
│ [ 조별리그 ] [ 토너먼트 ]  ← 탭            │  (sticky)   │
│                                            │  1 ✅ 🇫🇷   │
│  Group A          Group B    ...           │  ...        │
│  ┌ 경기카드 ─────┐                         │  8 ✅ 🇯🇵   │
│  │ 🇫🇷 [2]–[1] 🇲🇽 │  (스코어 +/− stepper)  │ ──────────  │
│  └───────────────┘                         │  9 ❌ ...   │
│  ┌ 조 순위표 ────┐                         │ 12 ❌ ...   │
│  │ 1 🇫🇷  3 0 0 ..│                         │             │
│  └───────────────┘                         │ 실시간 갱신 │
└──────────────────────────────────────────┴─────────────┘
```

- **조별리그 탭**: 12개 조 그리드. 각 조 = 경기 카드(스코어 +/− stepper) + 실시간 순위표.
- **토너먼트 탭**: 브래킷. R32부터 경기 클릭 → 승자 선택(승패만). 조별리그 결과에서 자동 시드.
- **Floating 3위 패널**: 12팀 랭킹, 상위 8 진출(✅)/탈락(❌). 스코어 변동 시 실시간 갱신.
  - 데스크탑: 우측 sticky 패널.
  - 모바일: 하단 시트(FAB로 토글).

## 5. 테마

- 키컬러 녹색. **라이트=짙은 녹색, 다크=밝은 녹색.**
- CSS 변수 + Tailwind + shadcn 토큰. `next-themes`로 토글, `prefers-color-scheme` 기본값.

---

## 6. 리셋 & 엣지케이스

- **"실제 결과로 초기화"**: 모든 스코어/승자 → seed 디폴트 복원.
- (옵션) **"전체 비우기"**: 모든 입력 제거.
- 미입력 경기 = "미진행", 순위에서 제외.
- 32강 시드는 72경기 모두 입력돼야 완전 활성화. 부분 입력 시 미리보기 표시.

## 8. 데이터 시딩 절차

`scripts/seed-worldcup.ts`:
1. 외부 소스(WebFetch/스크랩 또는 API)에서 2026 WC 조편성·일정·현재 결과 fetch.
2. `data/worldcup-2026.json` 생성 후 커밋.
3. 앱은 런타임에 이 JSON만 읽음 (외부 API 런타임 의존 없음).

### 리스크
1. **실데이터 fetch**: 컷오프(2026.1) 이후인 6월 경기 결과는 웹에서 긁어와야 함. 웹 접근 가능 시 현재 결과까지, 불가 시 조편성·일정만 시드하고 결과는 비워둠.
2. **3위 배정 매트릭스**: 2026 포맷의 R32 3위 배정 공식표가 가장 까다로운 구현 포인트. 공식 구조를 정확히 반영.

## 9. 컴포넌트 경계 (격리/명확성)

| 단위 | 역할 | 의존 |
|------|------|------|
| `data/worldcup-2026.json` | 정적 시드 데이터 | 없음 |
| `lib/standings.ts` | 조 순위·3위 랭킹 계산 (순수) | seed 타입 |
| `lib/knockout.ts` | R32 시드·브래킷 전파 (순수) | standings 결과 |
| `store/` (Zustand) | 런타임 스코어/승자 상태 + localStorage | lib |
| `components/group/*` | 조별리그 탭 UI (경기카드, 순위표) | store, lib |
| `components/knockout/*` | 토너먼트 브래킷 UI | store, lib |
| `components/ThirdPlacePanel` | Floating 3위 랭킹 | store, lib |
| `components/theme/*` | 테마 토글 | next-themes |

각 `lib/*`는 UI 없이 테스트 가능. UI는 lib 결과를 렌더만.
